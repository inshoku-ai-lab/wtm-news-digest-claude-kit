# WTM News Digest — ワークフロー定義

Telegram エクスポート（英語チャネル = `en.txt`、日本語訳チャネル = `jp.txt`、
任意で `jp.pdf`）を横断統合し、編集者がそのまま使える 3 種の成果物を生成する。

## 絶対原則（全工程で厳守）

- **再翻訳・創作・推論を一切混ぜない**。英語原文は `en.txt`、日本語訳は `jp.txt` から
  ありのまま抽出する。ソースに無い項目は必ず **`[データ不在]`** と書く。
- **機械処理は決定論的に**（抽出・ID 紐付け・反応数集計・並べ替え）。
  AI 判断は要約・タイトル・分類・クラスタリング・記事執筆に限定する。
- 大量出力は分割し、各工程の終わりでユーザー確認を待つ。
- 抽出結果の件数・重複・欠落を自己検算し、一言で報告する。

## データ構造の要点（紐付け原理）

- **1 投稿 = 1 単位**。jp.txt 側で `forwarded_from` が `WeTheMedia🇯🇵日本語訳️️️` の
  メッセージが「翻訳アンカー」となり、これが投稿の単位を決める。
- 翻訳アンカーの本文中にある `text_link` の href（`https://t.me/WeTheMedia/<en_id>`）を
  使って、`en.txt` 側の該当投稿と一意に紐付ける。
- **リアクション総数・メディア種別・動画尺・英語原文は en.txt が正本**。
- **投稿 URL・投稿日時・日本語訳は jp.txt が正本**。投稿 URL は翻訳アンカー内の
  `🔗` の text_link の href（`https://t.me/WTM_deepl_jp/XXXXX`）。
- **画像 OCR・動画字幕**は jp.txt 内で、翻訳アンカーの直前の「英語転送メッセージ」
  の id を `reply_to_message_id` として指す返信メッセージ群にある。

## 除外ルール

- **付属メディア**（1 投稿に複数の画像・動画が添付されている場合の 2 枚目以降）は、
  en.txt 内でリアクションを持たない補助メッセージとして現れる。**除外する**。
- **自己宣伝 URL** は外部参照リンクから除外する：
  - `SGTnewsNetwork` を含む URL
  - `disclosetv`（転送元が Disclose.tv の場合）
  - `WatcherGuru`（転送元が Watcher Guru の場合）
  - `MyLordBebo`（転送元が MyLordBebo の場合）
  - `t.me/WeTheMedia` のトップページ URL
  - `t.me/+` で始まる招待リンク
  - `t.me/<単一チャネル名>` の形式のチャネル URL

## 既知の落とし穴

- **en.txt の末尾破損**：Telegram エクスポートの `result.json` を切り出した際に、
  末尾が `]\n}` で閉じられていないことがある。JSON パース失敗時は、末尾に `]\n}` を
  追記して再パースを試みる。
- **翻訳アンカーの `forwarded_from` 表記**：末尾に不可視文字（絵文字結合の
  variation selector など）が含まれるため、完全一致でマッチングする際は文字列を
  そのまま扱う。部分一致では意図しないメッセージがヒットする恐れがある。

## 実行フロー

### 工程 1：抽出・構造化

`en.txt` と `jp.txt` を読み込み、以下のロジックで投稿を抽出する。
コード実行環境（bash / python）が使える場合はスクリプトで、
使えない場合は Claude が直接論理を追って処理する。

**投稿の抽出手順：**

1. `jp.txt` を JSON パース。全メッセージから翻訳アンカー
   （`forwarded_from == "WeTheMedia🇯🇵日本語訳️️️"`）を抽出。
2. 各アンカー内の text_link から `t.me/WeTheMedia/<en_id>` を取り出し、
   `en_id` を得る。これで en.txt との紐付けができる。
3. `en.txt` を JSON パース（末尾破損時は `]\n}` を追記）。
4. 各 `en_id` に対応する en.txt メッセージから、リアクション・メディア・
   英語原文・動画尺を取得。リアクションを持たない補助メッセージ（付属メディア）は
   除外。
5. jp.txt 内の翻訳アンカーの直前メッセージ id を `reply_to_message_id` として持つ
   返信メッセージから、画像 OCR と動画字幕を取得。

**各投稿について以下の既存項目を Markdown で出力する：**

```
## 投稿番号 <連番>
- **投稿日時**: 日本 <値>, NY <値>, CEST <値>
- **参照リンク**:
    - 投稿URL: <jp.txt 側の 🔗 の href>
    - 引用メディア: <画像 / 動画（N秒） / GIF / 無し>
    - 外部参照リンク: <自己宣伝を除いた外部 URL、カンマ区切り、なければ 無し>
- **反応分析**:
    - リアクション総数: <N>（絵文字1: N1 / 絵文字2: N2 / ...）
- **テキスト・ログ（投稿内容の抽出）**:
    - 英語版原文:
> <英語原文の各行を > プレフィックス付きで転記>
    - 日本語版既訳:
> <jp.txt から抽出した翻訳本文を > プレフィックス付きで転記>
- **ビジュアル解析**:
    - 画像OCR/内容説明: <あれば > プレフィックス付きで転記、なければ 無し>
    - 動画内容抽出: <あれば > プレフィックス付きで転記、なければ 無し>
```

抽出後、**総投稿数と対象期間を報告**する。

### 工程 2：追加分析 4 項目の付与

各投稿に以下 4 項目を分析・付与する：

- **反応レベル**（1〜10）：リアクション総数の相対値と感情パターンから判断。
  参考の目安として、総数 100〜200 前後を 1〜2、900 前後を 10 とする
  （データセット全体の分布に応じて調整）。数値の後ろに感情パターンの短い注記を添える。
- **要約**（約 200 字）：日本語で、事実情報のみ。関連する他の投稿への言及も可。
  テキストが極端に少なく要約困難な投稿は「要約しない」と明記。
- **投稿タイトル**（10〜40 字）：ニュースサイトの見出しに使える表現。
- **投稿ジャンル**：以下の固定 10 種から 1 つを選択：
  1. 法と司法
  2. 米国内犯罪
  3. 米国政治
  4. 米国と他国の関係
  5. 他国の情勢
  6. 健康・医学
  7. 技術・科学・経済
  8. 社会問題・人権・環境
  9. 代替的視点・理論
  10. その他

**配慮調整が必要な投稿**（ヘイト表現・暴力扇動・自傷示唆・粗野な表現などを含む）は、
英語原文・日本語既訳・OCR はソースのまま転記した上で、要約欄で「記事化不適」と
明示する。事実情報の抽出は継続する。

### 工程 3：メインニュース選定

全投稿を横断し、**同一の固有名詞・事件・トピック**を共有する投稿を束ねる。
件数の多い順に上位 **最大 5 件** を選定する。各トピックについて：

- トピックの通し番号（①〜⑤）と名称
- 言及投稿の一覧（投稿番号のみ）
- 約 300 字の解説（どのような投稿群であるか、どのような事実が浮かび上がるか）

分類済み件数と未分類件数を検算し報告する。

### 工程 4：解説記事＋外部リンク＋関連投稿

5 トピックそれぞれについて、以下を執筆する。

**本文執筆規約：**
- 分量：600〜800 字
- 品質：ニュースサイト掲載レベル。AI らしい定型表現（「〜と言えるでしょう」の
  多用など）を避け、人間の記者が書いた自然な文体
- 立場：中道保守。事実ベース。党派的断定は避け、必要に応じて帰属表現
  （「〜と◯◯氏は述べた」）を使う
- 末尾：本文の最後に「AI による解説」と「コメント」の 2 行を必ず記載

**関連外部記事：**
- 3 件。ニュースサイトのカテゴリ URL ではなく、特定の個別記事の URL
- 実在検証を **2 回** 実施：
  1. web_search で該当記事を検索し、本文スニペットで内容を確認
  2. web_fetch または別の切り口の web_search で同じ記事の存在を再確認
  3. robots.txt でクロール制限のあるサイト（NPR など）は、検索結果に本文が
     現れれば実在確認とみなす
- 各記事に「補強内容の説明」を短く添える
- 投稿内に引用された外部記事がなく、web 検索でも見つからない場合は、
  同等の内容を報じる主流メディアの個別記事で補う
- どうしても見つからない場合は省略可

**関連投稿：**
- 該当するすべての投稿を、タイトルと URL のみで列挙。省略不可
- タイトルと URL の間は改行 2 つ
- 投稿ユニット間は改行 3 つ
- 「タイトル：」「URL：」などのラベルは付けない

### 工程 5：ジャンル別サブリスト

工程 3 でメインニュース 5 トピックに使用した投稿を除外し、残りを固定 10 ジャンルに
分類し、各ジャンル内は反応数の降順で並べる。

**フォーマット：**
- ジャンル名は数字（「1.」など）を省いて記載
- 投稿はタイトルと URL のみ
- タイトルと URL の間は改行 2 つ
- 投稿ユニット間は改行 3 つ
- ジャンル間は空行 1 つで区切る

除外件数・出力件数・ジャンル別件数を検算し報告する。

## 応答スタイル

- すべて日本語で応答する
- 各工程の終わりに次のアクションを提示し、ユーザー確認を待つ
- 大量出力は工程ごとに分割
- 抽出データや分析結果に自信が持てない箇所は `[データ不在]` または要約欄で明示
- 推測で埋めない
